Учебный проект 
по автоматизации ETL-процесса и развертыванию базы данных на выделенном сервере.

Цель проекта — отработка навыков развертывания базы данных на удаленном сервере и настройки автоматизированных ETL-процессов.

Проект решает задачу регулярной загрузки данных в базу данных в PostgreSQL 
Он состоит из двух основных скриптов:

•	`generate_data.py` — генерирует случайные данные и сохраняет их в папку `/data` в      виде CSV-файлов. 
Шаблон названия фалов: дата_”номер магазина”_”номер кассы”.csv. 
Структура файла: численно-буквенный идентификатор чека, название товара, категория товара, кол-во товара в чеке, цена одной позиции без учета скидки, скидка на эту позицию

•	`run.py` — забирает CSV-файлы из папки `/data` и загружает их в таблицу sales в базу данных `training_deploy`. Создает соответствующие поля, также формирует новые поля для информации из названия файла (дата, номер магазина, номер кассы).

Важно: Скрипт настроен так, чтобы автоматически пропускать работу по воскресеньям (как в коде Python, так и в расписании Cron)

Установка и запуск
Шаги для разворачивания проекта на сервере (Ubuntu/Debian)
1.	Проверить наличие нужных программ: python3, pip, psql
   
2.	Настроить виртуальное окружение (делается при каждом новом подключении)
apt install python3.12-venv
python3 -m venv venv
source venv/bin/activate

3.	Клонировать репозиторий.
a.	git clone –b master https://github.com/guzel-sa/deploy.git
b.	cd deploy

4.	Установить зависимости
pip install -r requirements.txt

5.	Настроить PostgreSQL
   
        su – postgres  \\\\ psql –U postgres \\\\ ALTER USER postgres WITH PASSWORD ‘*****’; \\\\ exit \\\\ su \\\\ Ввести пароль к серверу

6.	Отредактировать конфигурацию (config.ini)
При необходимости изменить SALES_PATH и пароль к базе данных

7.	Запустить PostgreSQL и создать базу данных training_deploy (используя DBeaver, подключение через ssh). Таблицы можно создать, используя скрипт DDL или они сформируются сами при запуске run.py
   
8.	Запустите вручную (для теста)
python3 generate_data.py
python3 run.py

9.	Автоматизация (Cron)
EDITOR=nano crontab -e

        Для ежедневного автоматического запуска в настройках Cron используется следующая команда:

        # Генерация данных в 07:01 (кроме воскресений)
        
        01 07 * * 1-6 cd /root/deploy && /root/venv/bin/python generate_data.py 
        
        # Загрузка в БД в 08:01 (кроме воскресений)
        
        01 08 * * 1-6 cd /root/deploy && /root/venv/bin/python run.py
